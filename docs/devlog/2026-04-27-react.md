#### ✅ Молчаливая ошибка при скачивании (две итерации)

**Итерация 1:** Добавлен `toast.error`, но с некорректной проверкой:

```ts
// НЕВЕРНО: "response" in err всегда true для AxiosError (ключ присутствует,
// даже если значение undefined)
if (!(err instanceof Error && "response" in err)) {
  toast.error("Не удалось скачать файл")
}
```

**Итерация 2 (финальная):** Исправлено через `axios.isAxiosError` + проверку значения, а не наличия ключа:

```ts
// ВЕРНО
import axios from "axios"

if (!axios.isAxiosError(err) || err.response == null) {
  toast.error("Не удалось скачать файл")
}
```

Матрица поведения:
| Ситуация | `isAxiosError` | `err.response` | Тост |
|---|---|---|---|
| Сетевая ошибка / CORS / DNS | `true` | `null` / `undefined` | ✅ показываем |
| HTTP 4xx / 5xx | `true` | объект с `status` | ❌ interceptor уже показал |
| Не Axios (TypeError и т.д.) | `false` | — | ✅ показываем |

#### 🚫 `h-100 w-100` / `h-75 w-75` — false positive

Copilot заявил, что классы не сгенерируются, т.к. шкала Tailwind v3 заканчивается на 96.  
Проект использует **Tailwind v4.2.1**, где `w-<number>` генерируется для любого числа через `calc(var(--spacing) * N)`. Классы корректны, изменений нет.

#### ⚠️ N+1 polling задач — tech debt

`DocumentAnonymizationRow` создаёт отдельный `useQuery` с polling 3 с на каждую строку. При большом числе документов это масштабируется плохо.

Требует батч-эндпойнта на бэкенде (`GET /tasks?document_ids=...`) или рефакторинга polling на уровень страницы. Зафиксировано в `TECH_DEBT.md` как **TD-001**.

### Второй раунд замечаний

#### ✅ Interceptor coverage — `toast` для 404/409/422

Предыдущая проверка `err.response == null` пропускала HTTP-коды вне зоны ответственности interceptor'а (400/403/5xx). Коды 404, 409, 422 молчали.

Финальная логика — явный список кодов, обрабатываемых interceptor'ом:

```ts
const status = axios.isAxiosError(err) ? err.response?.status : undefined
const handledByInterceptor =
  status === 400 || status === 403 || (status != null && status >= 500)
if (!handledByInterceptor) {
  toast.error("Не удалось скачать файл")
}
```

Та же логика применена в `startMutation.onError`.

#### ✅ `latestByModule(m: string)` → `m: TaskModule`

Параметр `m` принимал произвольную строку — опечатки в имени модуля проходили молча. Тип ужесточён до `TaskModule` — ошибки теперь ловит компилятор.

#### ✅ Retry всегда запускал `"convert"`

`startMutation.mutationFn` был захардкожен на `tasksApi.start(doc.id, "convert")`. Кнопка "Повторить" при падении anonymize перезапускала convert вместо anonymize.

Исправление: `mutationFn` принимает `module: TaskModule`; добавлена переменная `failedModule`:

```ts
const failedModule: TaskModule | null =
  convertStatus === "failed" ? "convert" :
  anonymizeStatus === "failed" ? "anonymize" :
  null
```

Кнопки:
- "Запустить" → `startMutation.mutate("convert")`
- "Повторить" → `startMutation.mutate(failedModule)` (disabled если `null`)

#### ✅ `React.ReactNode` в `StageChip`

Обнаружен второй экземпляр `React.ReactNode` в `styles` внутри `StageChip` — `React` namespace в этом файле не импортирован. Заменено на `ReactNode`.

#### ✅ Named props interfaces

`EntityBadge` и `DownloadButton` использовали inline-типы в сигнатуре. Добавлены `EntityBadgeProps` и `DownloadButtonProps` для консистентности с остальными компонентами файла.
