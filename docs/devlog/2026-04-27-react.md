### Замечания и решения

#### ✅ `React.ReactNode` → именованный импорт

`icon: React.ReactNode` в `DownloadButton` заменён на `icon: ReactNode` с явным `import type { ReactNode } from "react"`. Консистентность с остальными типами в файле.

#### ✅ Гонка состояний: кнопка "Запустить" при загрузке задач

Кнопка отображалась сразу (`!hasStarted`), не дожидаясь загрузки списка задач. Если задачи ещё не получены, `hasStarted` ложно `false`, и пользователь видел кнопку, которой не должно быть.

```tsx
// ДО
{!hasStarted && (
  <Button disabled={startMutation.isPending}>

// ПОСЛЕ
{!hasStarted && !isLoadingTasks && (
  <Button disabled={startMutation.isPending || isLoadingTasks}>
```

#### ✅ Молчаливая ошибка при скачивании (два итерации)

**Итерация 1:** Добавлен `toast.error`, но с некорректной проверкой:

```ts
// НЕВЕРНО: "response" in err всегда true для AxiosError (ключ присутствует,
// даже если значение undefined)
if (!(err instanceof Error && "response" in err)) {
  toast.error("Не удалось скачать файл")
}
```

**Итерация 2 (финальная):** Исправлено через `axios.isAxiosError` + проверку значения, а не наличия ключа:

```ts
// ВЕРНО
import axios from "axios"

if (!axios.isAxiosError(err) || err.response == null) {
  toast.error("Не удалось скачать файл")
}
```

Матрица поведения:
| Ситуация | `isAxiosError` | `err.response` | Тост |
|---|---|---|---|
| Сетевая ошибка / CORS / DNS | `true` | `null` / `undefined` | ✅ показываем |
| HTTP 4xx / 5xx | `true` | объект с `status` | ❌ interceptor уже показал |
| Не Axios (TypeError и т.д.) | `false` | — | ✅ показываем |

#### 🚫 `h-100 w-100` / `h-75 w-75` — false positive

Copilot заявил, что классы не сгенерируются, т.к. шкала Tailwind v3 заканчивается на 96.  
Проект использует **Tailwind v4.2.1**, где `w-<number>` генерируется для любого числа через `calc(var(--spacing) * N)`. Классы корректны, изменений нет.

#### ⚠️ N+1 polling задач — tech debt

`DocumentAnonymizationRow` создаёт отдельный `useQuery` с polling 3 с на каждую строку. При большом числе документов это масштабируется плохо.

Требует батч-эндпойнта на бэкенде (`GET /tasks?document_ids=...`) или рефакторинга polling на уровень страницы. Зафиксировано в `TECH_DEBT.md` как **TD-001**.
