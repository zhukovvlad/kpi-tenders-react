# Дневник разработки — react-kpi-tenders

## 2026-04-15 — День 1: Scaffolding, shadcn/ui, роутинг, лендинг, дашборд

### Что сделано

**1. Инициализация проекта**
- `npx shadcn@latest init -t vite` — создан Vite + React 19 + TypeScript проект
- Preset: **Nova** (Lucide + Geist Variable)
- Component library: **Radix UI**
- Tailwind CSS v4 через `@tailwindcss/vite` (vite-плагин, без `tailwind.config.js`)
- Path alias `@/*` → `./src/*` настроен в `vite.config.ts` и `tsconfig.app.json`

**2. Дополнительные зависимости**
- `react-router-dom@7` — клиентский роутинг
- `axios@1` — HTTP-клиент
- `react-dropzone@15` — drag-and-drop загрузка файлов

**3. Структура папок**
```
src/
├── components/
│   ├── ui/               ← shadcn компоненты (button и др.)
│   ├── GlassCard.tsx
│   ├── Header.tsx
│   ├── Hero.tsx
│   └── ProtectedRoute.tsx
├── context/
│   └── AuthContext.tsx
├── hooks/
│   └── useAuth.ts
├── pages/
│   ├── LandingPage.tsx
│   └── DashboardPage.tsx
├── services/api/
│   └── client.ts
├── lib/utils.ts
├── App.tsx
├── main.tsx
└── index.css
```

**4. API-клиент (`src/services/api/client.ts`)**
- `axios.create` с `baseURL: VITE_API_URL ?? "/api"` и `withCredentials: true`
- Токены **не хранятся в localStorage** — работа полностью через HttpOnly cookies (в пару к бэкенду `go-kpi-tenders`)
- Response interceptor: при `401` автоматически редиректит на `/login`

**5. AuthContext (`src/context/AuthContext.tsx`)**
- Стаб: `isAuthenticated: boolean` + методы `login()` / `logout()`
- По умолчанию `false`; при интеграции с бэкендом заменить `useState` на вызов `GET /api/v1/me` в `useEffect`

**6. Компоненты**
- `GlassCard` — `backdrop-blur-md`, `bg-white/5`, `border border-white/10`, `rounded-2xl`
- `Header` — фиксированный (`fixed top-0`), прозрачный, `backdrop-blur-sm`; кнопки Login и Request Access
- `Hero` — фон `bg-[#020617]`, три абстрактных сферы с `blur-[120px]` (фиолетовый/индиго), заголовок "Constructing Life", сабтайтл "Space By Space", кнопка "Get Started" со свечением через `box-shadow`
- `ProtectedRoute` — редиректит на `/` если `isAuthenticated === false`

**7. Страницы**
- `LandingPage` (`/`) — Header + Hero
- `DashboardPage` (`/dashboard`) — защищён ProtectedRoute:
  - Верхняя навигация с иконками Bell, User, LogOut
  - Drag-and-drop зона (`react-dropzone`): принимает PDF, DOC, DOCX
  - Таблица документов: Name, Size, Uploaded, Status (badge: ready / processing / error)
  - Stub-данные из локального стейта; загруженные через дропзону файлы добавляются в стейт со статусом `processing`

**8. Роутинг (`App.tsx`)**
```
/            → LandingPage
/dashboard   → ProtectedRoute → DashboardPage
```
- `AuthProvider` оборачивает `BrowserRouter` на верхнем уровне
- `ThemeProvider` (shadcn) подключён в `main.tsx`
