# Дневник разработки — react-kpi-tenders

## 2026-05-08 — День 11: Подключение к реальному бэкенду

### Контекст

До этой сессии фронтенд работал целиком на моках (`VITE_USE_MOCKS` не задан →
`USE_MOCKS = true`). Задача — подключить авторизацию к живому Go-бэкенду
(`go-kpi-tenders`, порт 8080) и устранить крэш и 404-ошибки, которые проявились
после отключения мок-слоя.

---

### Что сделано

#### 1. Отключение мок-слоя

Создан `.env.local` с `VITE_USE_MOCKS=false`. Это единственное место управления
режимом — не нужно менять код.

#### 2. Маппинг `organization_id` → `org_id` в `auth.ts`

Бэкенд возвращает `GET /api/v1/auth/me` с полем `organization_id`, а фронтовый
тип `User` ожидает `org_id`. Добавлен интерфейс `MeResponse` и явный маппинг:

```ts
const r = await apiClient.get<MeResponse>('/api/v1/auth/me')
return {
  ...r.data,
  org_id: r.data.organization_id,
  is_active: r.data.is_active,
}
```

Без этого `user.org_id` везде был `undefined`.

#### 3. Авторефреш токена в `client.ts`

Добавлен interceptor на 401: делает `POST /api/v1/auth/refresh`, ставит в очередь
параллельные запросы, повторяет оригинальный запрос после успешного рефреша.
Флаг `_retried` на `InternalAxiosRequestConfig` предотвращает бесконечный цикл.
Если рефреш не удался — позволяет 401 дойти до `AuthContext` (навигация на `/`).

#### 4. Исправление URL-адресов сервисов

Бэкенд не совпадал с URL-адресами, которые вызывали сервисы в dev-режиме:

| Сервис | Было | Стало |
|--------|------|-------|
| `sitesApi.listForDashboard` | `GET /api/v1/sites?aggregate=true&roots=true` | `GET /api/v1/sites/root` |
| `sitesApi.listChildren` | `GET /api/v1/sites?aggregate=true&parent_id=…` | `GET /api/v1/sites/:id/children` |
| `comparisonsApi.list` | `GET /api/v1/comparisons` | `GET /api/v1/comparison-sessions` |
| `comparisonsApi.save` | `POST /api/v1/comparisons` | `POST /api/v1/comparison-sessions` |
| `siteEventsApi.listForSite` | `GET /api/v1/sites/:id/events` | принудительный мок (см. ниже) |

#### 5. Нормализация ответа `SiteListItem` в `sites.ts`

Бэкенд возвращает `ConstructionSite` без агрегатных полей (`breadcrumbs`,
`contract_kinds`, `aggregate_status`, `extracted_count`, `inflation_pct`), которые
`SiteCard` читает напрямую — при этом падал с `Cannot read properties of undefined
(reading 'length')` на `site.breadcrumbs`.

Добавлена функция `normalizeSiteToListItem(site)`, которая дополняет ответ
безопасными дефолтами. Когда бэкенд начнёт возвращать агрегатные поля, они
автоматически вытеснят дефолты — маппинг не нужно будет трогать:

```ts
function normalizeSiteToListItem(site: ConstructionSite): SiteListItem {
  return {
    ...site,
    breadcrumbs: [],
    contract_kinds: [],
    aggregate_status: 'empty',
    extracted_count: 0,
    inflation_pct: null,
    last_activity_at: site.updated_at,
  }
}
```

#### 6. Принудительные моки для сервисов без бэкенда

Несколько сервисов вызывали эндпоинты, которых на бэкенде нет вовсе.
Вместо 404 — каждый сервис объявляет локальную константу `USE_*_MOCKS = true`
и остаётся на мок-данных до выравнивания API:

| Сервис | Причина |
|--------|---------|
| `keysApi` | Нет `/api/v1/extraction-keys` (есть `/contract-kinds`, `/file-roles`, но другая семантика) |
| `siteEventsApi` | Есть `/sites/:id/audit-log`, но схема `SiteAuditLog` не совпадает с `SiteEvent` |
| `comparisonsApi.listForSite` | Бэкенд не фильтрует сессии по `site_id` |
| `extractionApi.listForDocument` | Нет `/documents/:id/extraction-requests` |
| `extractionApi.answersForDocument` | Нет `/documents/:id/answers` |

#### 7. Документация расхождений для бэкенда

Создан файл `go-kpi-tenders/docs/frontend-api-gaps.md` — приоритизированный список
эндпоинтов, которые нужно добавить или исправить на бэкенде, с точными TypeScript-
контрактами ответов.

### Что не тронуто (и почему)

- `documentsApi.listBySite` — уже использует `GET /api/v1/documents?site_id=:id`,
  бэкенд поддерживает этот query-параметр ✅
- `extractionApi.initiate` / `.get` — эндпоинты `POST /documents/:id/extract` и
  `GET /extraction-requests/:id` существуют и работают ✅
- `tasksApi` — оставлен как есть (заглушка для будущей вкладки «Аудит») ✅
