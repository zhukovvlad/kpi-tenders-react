**2. `src/services/api/documents.ts` — API-сервис**

Методы для работы с документами:

| Метод | Endpoint | Описание |
|---|---|---|
| `upload(file)` | `POST /api/v1/documents/upload` | Загрузка через `multipart/form-data` |
| `list()` | `GET /api/v1/documents` | Список документов организации |
| `delete(id)` | `DELETE /api/v1/documents/:id` | Удаление документа |
| `getPresignedUrl(id, download?)` | `GET /api/v1/documents/:id/url` | Получение временной ссылки |

`getPresignedUrl` принимает флаг `download: boolean`:
- `false` (по умолчанию) → URL без `Content-Disposition` (браузер открывает inline)
- `true` → URL с `response-content-disposition=attachment` (форсированное скачивание)

**3. `src/pages/DocumentsPage.tsx` — страница документов**

Полностью реализована страница `/documents` с тремя ключевыми UX-решениями:

**Staged upload (загрузка с подтверждением)**
- `DropZone` принимает файлы через drag-and-drop или клик, но не загружает сразу
- Файлы попадают в очередь `staged: StagedFile[]` — пользователь видит список и может убрать лишние
- Кнопки «Очистить» и «Загрузить» появляются только при наличии файлов в очереди
- При клике «Загрузить» — последовательный `uploadMutation.mutate(file)` для каждого файла

**Просмотр и скачивание через presigned URL**
- Кнопка 👁 **Просмотр** — `getPresignedUrl(id, false)` → `window.open(..., "_blank")` (открывает inline)
- Кнопка ⬇ **Скачать** — `getPresignedUrl(id, true)` → программный клик на `<a download>` (форсирует сохранение)
- Оба действия имеют индикатор загрузки (spinner) и блокируют друг друга на время запроса
- Кнопки видны только при наведении на строку файла

**Удаление с подтверждением**
- Кнопка 🗑 открывает `Dialog` с именем файла
- Кнопки «Отмена» / «Удалить» — удаление только после явного подтверждения

**Пустое состояние и ошибки**: отдельные UI-состояния для загрузки, ошибки, пустого списка.

### Архитектурные решения

**Почему presigned URL, а не прямой MinIO URL**

Бакет `tenders` в MinIO приватный (AccessDenied при прямом обращении). Варианты:
1. Сделать бакет публичным — небезопасно, любой со ссылкой получит доступ
2. **Presigned URL через бэкенд** — выбран как правильный. Бэкенд проверяет `org_id` из JWT, затем генерирует временную ссылку (TTL 15 мин) через `PresignedGetObject` minio-go SDK
3. Proxy через бэкенд — наиболее безопасный, но нагружает сервер

**Безопасность (бэкенд — TODO)**

Эндпоинт `GET /api/v1/documents/:id/url` на бэкенде должен:
- Загружать документ из БД по `:id`
- Проверять `document.organization_id == org_id` из JWT → иначе 403
- Разбивать `storage_path` на `bucket` + `objectName` (`strings.SplitN`)
- Генерировать presigned URL через `PresignedGetObject` с `reqParams`:
  - `download=true` → `response-content-disposition: attachment; filename="..."`
  - `download=false` → пустой `reqParams` (inline)
