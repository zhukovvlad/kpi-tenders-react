# Дневник разработки — react-kpi-tenders

## 2026-04-21 — День 2: Регистрация организации — API, контекст, модальное окно

### Что сделано

**1. `src/services/api/auth.ts` — метод регистрации**

Добавлены интерфейсы и метод:

```ts
export interface RegisterCredentials {
  name: string       // название организации (required)
  inn?: string       // ИНН 10 цифр (optional)
  email: string      // email первого admin-пользователя
  password: string   // min 8 символов
  full_name: string  // полное имя пользователя
}

export interface RegisterResponse {
  org_id: string
  user_id: string
  warning?: string   // присутствует если токены не сгенерировались — нужен ручной логин
}
```

`authApi.register` вызывает `POST /api/v1/auth/register` — публичный endpoint бэкенда, атомарно создаёт организацию + первого admin-пользователя и выдаёт HttpOnly auth-куки.

**2. `src/context/AuthContext.tsx` — метод `register`**

- `register(credentials)` добавлен в `AuthContextValue` и реализован в `AuthProvider`
- После успешного вызова устанавливает `isAuthenticated(true)` только если в ответе нет поля `warning` (штатный сценарий — куки выданы)
- Если `warning` присутствует (бэкенд вернул 201 но без токенов) — `isAuthenticated` не меняется, пользователь должен войти вручную

### Принятые решения

- **Модальное окно вместо отдельной страницы** — регистрация происходит не уходя с лендинга, оба окна переключаются без перезагрузки
- **`warning` в ответе регистрации** — бэкенд может вернуть 201 без токенов (сбой генерации JWT при уже созданных данных); в этом случае `isAuthenticated` не устанавливается и пользователь видит `LoginModal`
- **Управление состоянием в `LandingPage`** — оба модала живут на одном уровне, коллбеки переключения исключают одновременное открытие двух окон
