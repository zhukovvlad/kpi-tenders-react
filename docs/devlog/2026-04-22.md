# Дневник разработки — react-kpi-tenders

## 2026-04-22 — День 3: Сессия через `/auth/me`, TanStack Query, логгер

### Что сделано

**1. `src/types/auth.ts` — новый файл**

Выделен интерфейс пользователя:

```ts
export interface User {
  id: string
  org_id: string
  email: string
  full_name: string
  role: 'admin' | 'member'
}
```

**2. `src/lib/logger.ts` — централизованный логгер**

Обёртка над `console.*` с фильтрацией по окружению:

- `logger.info` — только в `development`
- `logger.warn` / `logger.error` — всегда

Заменяет прямые вызовы `console.log` в бизнес-логике и сервисах.

**3. `src/services/api/auth.ts` — метод `fetchMe`**

Удалён хак `checkSession`, который дёргал `/api/v1/documents` для проверки сессии. Добавлен корректный метод:

```ts
fetchMe: (): Promise<User> =>
  apiClient.get<User>('/api/v1/auth/me').then((res) => res.data),
```

**4. `src/services/api/client.ts` — response interceptor**

Централизованная обработка HTTP-ошибок:

| Статус | Действие |
|---|---|
| `403` | `toast.error('Доступ запрещён')` |
| `400` | `toast.error(message \|\| 'Некорректный запрос')` |
| `5xx` | `toast.error('Ошибка сервера. Попробуйте позже.')` |
| `401` | Не обрабатывается здесь — делегировано `AuthContext` во избежание redirect loop |

**5. `src/hooks/useUser.ts` — новый хук**

Хук на базе `useQuery` (@tanstack/react-query):

- `queryKey: ['user', 'me']`
- `staleTime: 5 минут` — исключает лишние запросы при переключении вкладок
- `retry`: никогда не повторяет при `401`, до 2 раз при других ошибках

**6. `src/context/AuthContext.tsx` — полный рефакторинг**

Удалён ручной `useState` + `useEffect` с вызовом `checkSession`. Контекст переведён на `useUser()`:

- `isAuthenticated` = `!!user` (данные из кеша TanStack Query)
- `isLoading` = флаг первичного запроса
- `user` добавлен в значение контекста
- При ошибке `401`: `navigate('/')` + toast если сообщение содержит «unavailable»
- Редирект защищён от петли: не срабатывает если уже на `/`
- `login()` после успеха вызывает `queryClient.invalidateQueries(['user', 'me'])` — инвалидирует кеш, запускает повторный `fetchMe`
- `logout()` вызывает `queryClient.clear()` + `navigate('/')`
- Логирование через `logger.info` / `logger.warn`

**7. `src/App.tsx` — провайдеры**

- Добавлен `QueryClientProvider` (обёртка самого верхнего уровня)
- `AuthProvider` перемещён **внутрь** `BrowserRouter` — это обязательно, так как `AuthContext` использует `useNavigate`
- Добавлен `<Toaster />` из `@/components/ui/sonner` (shadcn-компонент)

**8. Зависимости**

```bash
npm install @tanstack/react-query
npx shadcn@latest add sonner
```

`sonner` (npm-пакет) был уже установлен ранее как транзитивная зависимость; shadcn CLI добавил обёртку `src/components/ui/sonner.tsx`.

### Принятые решения

- **`useUser` вместо `useEffect` в контексте** — сессия теперь кешируется TanStack Query; повторные mount-ы компонентов не генерируют лишние запросы
- **`401` не обрабатывается в глобальном interceptor** — обработка в `AuthContext` позволяет учитывать текущий маршрут и избегать redirect loop при уже открытом `/login`
- **`AuthProvider` внутри `BrowserRouter`** — необходимо для доступа к `useNavigate`; `QueryClientProvider` остаётся снаружи, так как не зависит от роутера
- **`toast()` из `sonner` напрямую** — shadcn оборачивает только `Toaster`-компонент; `toast()` по документации shadcn/sonner импортируется из пакета `sonner`
